## Основной поток: Создание позиции

1. Пользователь открывает раздел «Справочники → Прайс-лист услуг».
2. Система отображает список активных позиций с колонками: Код, Наименование, Категория, Единица, Цена, Статус.
3. Пользователь нажимает «Добавить услугу».
4. Система отображает форму создания.
5. Пользователь заполняет поля:
   - **Код** — уникальный артикул (SRV-001, DIAG-01 и т.п.)
   - **Наименование** — например, «Выезд инженера», «Диагностика банкомата»
   - **Описание** — необязательно, текст
   - **Категория** — Ремонт / ТО / Диагностика / Выезд / Прочее
   - **Единица измерения** — шт / час / выезд / комплект
   - **Цена** — числовое поле
   - **Валюта** — RUB (по умолчанию)
6. Пользователь нажимает «Сохранить».
7. Система валидирует: код уникален, наименование не пустое, цена ≥ 0.
8. Система сохраняет позицию со статусом `is_active = true`.
9. Позиция отображается в списке.

## Альтернативные потоки

**АП-1: Редактирование позиции**
На шаге 2 пользователь нажимает «Редактировать» у нужной позиции.
Система открывает форму с заполненными полями.
Пользователь изменяет поля и нажимает «Сохранить».
Система обновляет позицию. Ранее сохранённые акты и счета не пересчитываются (BR-P-003).

**АП-2: Деактивация позиции**
Пользователь нажимает «Деактивировать» у позиции.
Система устанавливает `is_active = false`.
Позиция исчезает из выпадающих списков в актах и счетах, но остаётся в исторических документах.

**АП-3: Просмотр деактивированных**
Пользователь включает фильтр «Показать неактивные».
Система отображает все позиции включая деактивированные.
Рядом с деактивированными отображается метка «Неактивна» и кнопка «Активировать».

**АП-4: Попытка удаления позиции с историей**
Позиция используется в актах или счетах.
→ Система запрещает удаление (BR-P-006): «Позиция используется в документах. Деактивируйте её».

---

## Исключения

**ИП-1: Дублирование кода**
→ Система блокирует: «Код [X] уже используется. Введите уникальный код».

**ИП-2: Цена отрицательная**
→ Система блокирует: «Цена не может быть отрицательной».

**ИП-3: Пустое наименование**
→ Система блокирует: «Введите наименование услуги».

## Acceptance Criteria

```gherkin
Feature: Справочник услуг

  Scenario: Успешное создание позиции
    Given пользователь с ролью svc_mgr открыл «Прайс-лист услуг»
    When заполняет: Код=SRV-001, Наименование=«Диагностика», Категория=Диагностика, Ед=шт, Цена=1500
    And нажимает «Сохранить»
    Then позиция SRV-001 отображается в списке со статусом «Активна»

  Scenario: Блокировка дублирующего кода
    Given уже существует позиция с кодом SRV-001
    When пользователь создаёт новую позицию с кодом SRV-001
    Then система показывает «Код SRV-001 уже используется»
    And сохранение блокируется

  Scenario: Деактивация позиции без истории
    Given позиция SRV-002 не используется ни в одном документе
    When пользователь нажимает «Деактивировать»
    Then позиция получает is_active = false
    And исчезает из выпадающих списков в форме акта

  Scenario: Запрет удаления позиции с историей
    Given позиция SRV-001 используется в акте #10
    When пользователь пытается удалить SRV-001
    Then система показывает «Позиция используется в документах. Деактивируйте её»
```
