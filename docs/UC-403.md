# UC-403: Создать счёт на основе акта выполненных работ

**Версия:** 1.0 | **Дата:** 07.04.2026 | **Статус:** В работе

---

## Паспорт

| Поле | Значение |
|---|---|
| **ID** | UC-403 |
| **Модуль** | Модуль 4: Выставление счетов |
| **Основной актор** | Менеджер / Руководитель сервиса |
| **Второстепенные акторы** | Система (копирование позиций) |
| **Связанные BR-F** | BR-F-400, BR-F-410 |
| **Связанные BR-R** | BR-P-004, BR-P-005 |
| **Приоритет** | Must |
| **Предусловия** | Акт выполненных работ сохранён (статус заявки ≥ «Выполнена»); пользователь авторизован |
| **Постусловия** | Счёт создан в статусе «Черновик» с позициями из акта |
| **Триггер** | Пользователь нажимает «Создать счёт» в карточке заявки или акта |

---

## Основной поток

1. Пользователь открывает карточку заявки с сохранённым актом.
2. Пользователь нажимает **«Создать счёт по акту»**.
3. Система проверяет: акт существует, заявка не гарантийная.
4. Система создаёт черновик счёта и автоматически заполняет:
   - **Клиент** — из заявки
   - **Привязка к заявке** — `ticket_id`
   - **Дата выставления** — сегодня
   - **Позиции** — копируются из строк акта:
     - Тип, наименование, service_id / part_id, количество, единица, **зафиксированная цена** из акта
5. Система открывает форму нового счёта с заполненными позициями (BR-F-410).
6. Пользователь проверяет позиции, при необходимости редактирует (меняет цену, удаляет/добавляет строки).
7. Пользователь устанавливает **срок оплаты**.
8. Пользователь нажимает **«Сохранить черновик»**.
9. Счёт сохранён. Система отображает ссылку: «Счёт #INV/… создан на основе акта заявки #…».

---

## Альтернативные потоки

**АП-1: Гарантийная заявка**
На шаге 3 система обнаруживает `is_warranty = true`.
→ Система показывает предупреждение: «Заявка гарантийная. Счёт клиенту не выставляется (BR-P-005)».
→ Кнопка «Создать счёт по акту» недоступна. Пользователь может создать счёт вручную через UC-401 с нулевой суммой при необходимости.

**АП-2: Акт без позиций**
Акт сохранён, но строк нет (описание только текстом).
→ Счёт создаётся без позиций. Система предупреждает: «Акт не содержит позиций. Добавьте строки вручную».

**АП-3: Счёт уже существует для этой заявки**
→ Система предупреждает: «По этой заявке уже создан счёт #INV-…. Создать ещё один?»
→ Пользователь подтверждает или отменяет.

---

## Исключения

**ИП-1: Акт не сохранён**
→ Кнопка «Создать счёт по акту» недоступна. Подсказка: «Сначала сохраните акт выполненных работ».

---

## Ключевые данные

Счёт наследует структуру из UC-401. Дополнительно:

| Поле | Значение при создании из акта |
|---|---|
| client_id | Из заявки |
| ticket_id | ID заявки |
| items[].unit_price | Цена из акта (зафиксированная) |
| items[].service_id / part_id | Из строк акта |

## Acceptance Criteria

```gherkin
Feature: Счёт на основе акта

  Scenario: Успешное создание счёта из акта
    Given заявка #10 с актом: 1× SRV-001 «Диагностика» по 1500 [системная валюта], 2× PART-007 «Ролик» по 500 [системная валюта]
    When пользователь нажимает «Создать счёт по акту»
    Then счёт создан в статусе «Черновик»
    And содержит 2 строки: SRV-001 (1500 [системная валюта]) и PART-007 × 2 (1000 [системная валюта])
    And total_amount = 2500 [системная валюта] (до НДС)

  Scenario: Блокировка для гарантийной заявки
    Given заявка #11 помечена как гарантийная (is_warranty = true)
    When пользователь открывает карточку заявки
    Then кнопка «Создать счёт по акту» недоступна
    And отображается подсказка «Заявка гарантийная. Счёт не выставляется»

  Scenario: Предупреждение о дублирующем счёте
    Given заявка #10 уже имеет счёт #INV/2026/04/001
    When пользователь нажимает «Создать счёт по акту» повторно
    Then система показывает «По этой заявке уже создан счёт #INV/2026/04/001. Создать ещё один?»
```
