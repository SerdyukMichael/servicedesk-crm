# UC-804: Просмотреть панель загрузки инженеров

**Версия:** 1.0 | **Дата:** 26.03.2026 | **Статус:** Согласован

---

## Паспорт

| Поле | Значение |
|---|---|
| **ID** | UC-804 |
| **Модуль** | Модуль 8: Учёт персонала |
| **Основной актор** | Руководитель сервиса |
| **Второстепенные акторы** | Администратор, Директор (только просмотр) |
| **Связанные BR-F** | BR-F-803, BR-F-804 |
| **Связанные BR-R** | — (панель только для чтения, аудит-лог не применяется) |
| **Приоритет** | Must |
| **Предусловия** | Пользователь авторизован с ролью **Руководитель сервиса**, **Администратор** или **Директор**; в системе есть хотя бы один инженер со статусом «Активен» |
| **Постусловия** | Нет изменений в данных — панель только для чтения. Переход к назначению заявки возможен из контекста панели |
| **Триггер** | Пользователь открывает раздел **«Загрузка инженеров»** или вызывает панель в момент назначения заявки (→ UC-901) |

---

## Основной поток

1. Пользователь переходит в раздел **«Загрузка инженеров»**.
2. Система отображает панель: таблицу всех активных инженеров с текущими показателями нагрузки.
3. Пользователь применяет фильтры при необходимости (см. АП-1, АП-2).
4. Пользователь анализирует загрузку, принимает решение о назначении или перераспределении заявок.
5. Пользователь переходит к нужной заявке (клик по счётчику или названию заявки) — система открывает карточку заявки (→ UC-901).

---

## Состав панели

**По каждому инженеру отображается:**

| Столбец | Описание |
|---|---|
| Имя / Фото | ФИО инженера, аватар |
| Статус | Активен / В отпуске / Недоступен |
| Заявок в работе | Количество заявок со статусами `assigned`, `in_progress` |
| Просрочено | Количество заявок, у которых нарушен SLA |
| Запланировано на сегодня | Заявки с датой выезда = сегодня |
| Компетенции | Краткий список моделей оборудования (иконки или теги) |
| Последняя активность | Время последнего обновления по любой из его заявок |

**Визуальная индикация нагрузки:**
- 🟢 0–3 заявки в работе — низкая загрузка
- 🟡 4–6 заявок — средняя загрузка
- 🔴 7+ заявок — высокая загрузка
- ⚫ В отпуске / Недоступен — назначение невозможно

---

## Альтернативные потоки

**АП-1: Фильтрация по компетенции**
Пользователь выбирает модель оборудования в фильтре.
Система показывает только инженеров, имеющих активную (не истёкшую) компетенцию по выбранной модели.
Используется при поиске исполнителя для конкретной заявки.

**АП-2: Фильтрация по статусу**
Пользователь выбирает фильтр: «Только доступные» / «В отпуске» / «Все».
По умолчанию: «Только доступные».

**АП-3: Раскрытие списка заявок инженера**
Пользователь кликает на счётчик **«Заявок в работе»** у конкретного инженера.
Система разворачивает под строкой инженера список его активных заявок: ID, тема, приоритет, статус, дедлайн SLA.
Пользователь может перейти в любую заявку кликом.

**АП-4: Вызов панели из контекста назначения (встроенный режим)**
При назначении заявки в UC-901 система показывает эту же панель, но:
- Предварительно применён фильтр по компетенции (модель из заявки)
- Инженеры с истёкшим сертификатом и статусом «В отпуске» скрыты
- Доступна кнопка **«Назначить»** прямо из строки инженера

---

## Исключения

**ИП-1: Нет доступных инженеров по выбранному фильтру**
Система отображает: *«Нет инженеров, соответствующих фильтру. Попробуйте изменить параметры или проверьте профили компетенций»*.

**ИП-2: Все инженеры в отпуске / недоступны**
Система отображает панель с пометкой ⚫ у всех инженеров + предупреждение: *«Нет доступных инженеров. Проверьте статусы сотрудников»*.

---

## Ключевые данные (только чтение, вычисляемые)

| Показатель | Источник | Формула |
|---|---|---|
| Заявок в работе | Таблица заявок | `COUNT(tickets WHERE assignee_id = X AND status IN ('assigned','in_progress'))` |
| Просрочено | Таблица заявок | `COUNT(...WHERE sla_deadline < NOW() AND status NOT IN ('resolved','closed'))` |
| Запланировано на сегодня | Таблица заявок | `COUNT(...WHERE scheduled_date = TODAY())` |
| Последняя активность | Таблица заявок | `MAX(updated_at WHERE assignee_id = X)` |

## Acceptance Criteria

```gherkin
Feature: Панель загрузки инженеров

  Scenario: Отображение панели с корректными счётчиками
    Given в системе есть инженер «Иванов» с 5 заявками в статусе in_progress
    And 2 из них с нарушенным SLA
    When Руководитель сервиса открывает панель загрузки
    Then строка «Иванов» показывает: Заявок в работе = 5, Просрочено = 2
    And индикатор нагрузки — 🟡 (средняя)

  Scenario: Фильтрация по компетенции скрывает нерелевантных инженеров
    Given инженер «Петров» имеет компетенцию по «Matica XID 580i»
    And инженер «Сидоров» не имеет компетенции по этой модели
    When пользователь фильтрует панель по модели «Matica XID 580i»
    Then «Петров» отображается в списке
    And «Сидоров» не отображается

  Scenario: Инженер в отпуске помечен как недоступный
    Given статус инженера «Кузнецов» = «В отпуске»
    When Руководитель сервиса открывает панель
    Then строка «Кузнецов» отображает индикатор ⚫ «В отпуске»

  Scenario: Раскрытие списка заявок инженера
    Given инженер «Иванов» имеет 3 активные заявки
    When пользователь кликает на счётчик «Заявок в работе» у «Иванова»
    Then под строкой разворачивается список 3 заявок с ID, темой, приоритетом и дедлайном SLA

  Scenario: Встроенный режим при назначении заявки
    Given создана заявка на оборудование «Matica XID 580i»
    When Руководитель сервиса открывает панель в контексте назначения
    Then фильтр по компетенции «Matica XID 580i» применён автоматически
    And инженеры с истёкшим сертификатом не отображаются
    And у каждой строки инженера есть кнопка «Назначить»

  Scenario: Пустая панель при отсутствии доступных инженеров
    Given все инженеры имеют статус «В отпуске»
    When пользователь применяет фильтр «Только доступные»
    Then система отображает сообщение «Нет инженеров, соответствующих фильтру»
```

## Примечания / Бизнес-правила

- **BR-F-803:** панель — только для чтения. Изменение данных через эту панель не предусмотрено.
- **BR-F-804:** логика фильтрации инженеров (компетенция + статус) единая — и на самостоятельной панели, и в режиме назначения заявки (→ UC-901).
- Данные панели — реального времени, без кэша: пересчитываются при каждом открытии.
- Панель не является аналитическим дашбордом (исторические тренды, отчёты по периодам — в Модуле 14). Здесь только **текущий срез**.
