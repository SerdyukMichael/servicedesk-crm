# UC-803: Управлять профилем компетенций инженера

**Версия:** 1.1 | **Дата:** 27.03.2026 | **Статус:** На проверке

---

## Паспорт

| Поле | Значение |
|---|---|
| **ID** | UC-803 |
| **Модуль** | Модуль 8: Учёт персонала |
| **Основной актор** | Руководитель сервиса |
| **Второстепенные акторы** | Администратор (те же права), Система (аудит-лог) |
| **Связанные BR-F** | BR-F-802, BR-F-804 |
| **Связанные BR-R** | BR-R-010 (аудит-лог) |
| **Приоритет** | Must |
| **Предусловия** | Пользователь авторизован с ролью **Руководитель сервиса** или **Администратор**; карточка сотрудника существует, статус — **Активен**; справочник моделей оборудования заполнен |
| **Постусловия** | Профиль компетенций сохранён; изменения немедленно влияют на фильтрацию инженеров при назначении заявок (→ UC-901) |
| **Триггер** | Руководитель сервиса открывает карточку инженера и переходит на вкладку **«Компетенции»** |

## Основной поток — Добавление компетенции

1. Руководитель сервиса открывает карточку инженера → вкладка **«Компетенции»**.
2. Система отображает текущий список сертифицированных моделей оборудования (пустой при первом заходе).
3. Руководитель сервиса нажимает **«+ Добавить модель»**.
4. Система отображает выпадающий список из справочника моделей оборудования.
5. Руководитель сервиса выбирает модель, заполняет:
   - Номер сертификата (опционально)
   - Срок действия сертификата — дата (опционально)
   - Дата последнего обучения — дата (опционально)
6. Руководитель сервиса нажимает **«Сохранить»**.
7. Система валидирует данные: если поля заполнены — проверяет формат даты и что дата последнего обучения не позже текущей.
8. Система добавляет запись в профиль компетенций, фиксирует в аудит-логе.
9. Система отображает обновлённый список компетенций.

---

## Основной поток — Редактирование компетенции

1. Руководитель сервиса открывает вкладку **«Компетенции»**, нажимает **«Редактировать»** у нужной строки.
2. Система переводит строку в режим редактирования.
3. Руководитель сервиса обновляет номер сертификата, срок действия или дату обучения (любое из полей опционально, можно очистить).
4. Нажимает **«Сохранить»**.
5. Система валидирует и сохраняет, фиксирует в аудит-логе (`action: competency_update`, `model`, старое/новое значение).

> **Примечание:** история предыдущих версий сертификата не ведётся. Актуальное состояние + аудит-лог изменений.

---

## Основной поток — Удаление компетенции

1. Руководитель сервиса нажимает **«Удалить»** у нужной строки.
2. Система показывает подтверждение: *«Удалить компетенцию по модели [Название]? Инженер перестанет появляться при назначении заявок на это оборудование»*.
3. Руководитель сервиса подтверждает.
4. Система применяет soft delete для записи компетенции, фиксирует в аудит-логе.
5. Система обновляет список.

---

## Альтернативные потоки

**АП-1: Истёкший сертификат**
Применяется только если поле `certificate_valid_until` заполнено и дата уже прошла.
Система автоматически отображает компетенции с истёкшим сроком с визуальной меткой ⚠️ **«Сертификат истёк»**.
Инженер с истёкшим сертификатом **не появляется** в списке при назначении заявки на данную модель (→ UC-901).
Если `certificate_valid_until` не заполнен — проверка срока не выполняется, инженер доступен для назначения.
Руководитель сервиса может обновить данные через редактирование.

**АП-2: Срок сертификата истекает в течение 30 дней**
Применяется только если поле `certificate_valid_until` заполнено.
Система отображает метку 🔔 **«Истекает через N дней»** рядом с компетенцией.
Никаких ограничений на назначение заявок — инженер по-прежнему доступен.

---

## Исключения

**ИП-1: Дублирование модели**
На шаге 7: инженер уже имеет компетенцию по выбранной модели.
→ Система блокирует: *«Компетенция по модели [Название] уже добавлена. Используйте редактирование для обновления данных»*.

**ИП-2: Срок действия сертификата в прошлом (при добавлении)**
На шаге 7: поле `certificate_valid_until` заполнено, и введённая дата уже прошла.
→ Система предупреждает: *«Срок действия сертификата уже истёк. Добавить как истёкший?»*. Администратор подтверждает — запись сохраняется с меткой ⚠️. Если поле не заполнено — предупреждение не показывается.

**ИП-3: Удаление единственной компетенции у инженера с открытыми заявками**
Система показывает предупреждение: *«У инженера есть N открытых заявок на эту модель оборудования. После удаления компетенции переназначение невозможно автоматически. Продолжить?»*
Удаление не блокируется — только предупреждение.

## Ключевые данные

| Поле | Обязательное | Тип | Ограничения |
|---|---|---|---|
| engineer_id | Да | FK → users | Сотрудник с ролью «Сервисный инженер» |
| equipment_model_id | Да | FK → equipment_models | Из справочника моделей |
| certificate_number | Нет | varchar(100) | NULL если не заполнено |
| certificate_valid_until | Нет | date | NULL если не заполнено; если заполнено — любая дата (включая прошлую — с предупреждением) |
| last_training_date | Нет | date | NULL если не заполнено; если заполнено — не позже текущей даты |
| is_deleted | Да | boolean | Soft delete, по умолчанию false |

> История предыдущих версий сертификата не хранится. Одна актуальная запись на пару (инженер, модель).

## Acceptance Criteria

```gherkin
Feature: Управление профилем компетенций инженера

  Scenario: Успешное добавление компетенции без сертификата
    Given Руководитель сервиса открыл профиль инженера, вкладку «Компетенции»
    When он добавляет модель «Matica XID 580i», оставляя поля сертификата пустыми
    And нажимает «Сохранить»
    Then компетенция появляется в списке
    And инженер отображается при назначении заявок на модель «Matica XID 580i»
    And в аудит-логе: action=competency_add, model=Matica XID 580i

  Scenario: Успешное добавление компетенции с заполненным сертификатом
    Given Руководитель сервиса открыл профиль инженера, вкладку «Компетенции»
    When он добавляет модель «Matica XID 580i» с номером сертификата «CERT-001»,
         сроком действия через 1 год и датой обучения сегодня
    And нажимает «Сохранить»
    Then компетенция появляется в списке с данными сертификата
    And инженер отображается при назначении заявок на модель «Matica XID 580i»

  Scenario: Блокировка дублирования модели
    Given инженер уже имеет компетенцию по модели «Matica XID 580i»
    When Руководитель сервиса пытается добавить ту же модель повторно
    Then система показывает ошибку «Компетенция по модели Matica XID 580i уже добавлена»
    And сохранение блокируется

  Scenario: Инженер с истёкшим сертификатом не отображается при назначении
    Given у инженера есть компетенция по модели «Matica XID 580i»
    And certificate_valid_until заполнен и истёк вчера
    When Руководитель сервиса открывает список инженеров для назначения заявки на «Matica XID 580i»
    Then инженер НЕ присутствует в списке доступных
    And в профиле компетенций отображается метка «Сертификат истёк»

  Scenario: Инженер без заполненного certificate_valid_until доступен для назначения
    Given у инженера есть компетенция по модели «Matica XID 580i»
    And поле certificate_valid_until не заполнено
    When Руководитель сервиса открывает список инженеров для назначения заявки на «Matica XID 580i»
    Then инженер присутствует в списке доступных

  Scenario: Инженер с сертификатом, истекающим через 30 дней, доступен для назначения
    Given у инженера есть компетенция по модели «Matica XID 580i»
    And certificate_valid_until заполнен и истекает через 20 дней
    When Руководитель сервиса открывает список инженеров для назначения заявки
    Then инженер присутствует в списке
    And рядом с его именем отображается метка «Истекает через 20 дней»

  Scenario: Предупреждение при удалении компетенции с открытыми заявками
    Given у инженера есть 2 открытые заявки на модель «Matica XID 580i»
    When Руководитель сервиса нажимает «Удалить» для этой компетенции
    Then система показывает предупреждение «У инженера есть 2 открытых заявки на эту модель»
    And при подтверждении компетенция удаляется (soft delete)
```

## Примечания / Бизнес-правила

- **BR-F-802:** профиль компетенций ведётся только для сотрудников с ролью «Сервисный инженер». Для других ролей вкладка «Компетенции» не отображается.
- **BR-F-804:** при назначении заявки система фильтрует инженеров по двум условиям одновременно: (1) есть компетенция на модель оборудования из заявки, И если `certificate_valid_until` заполнен — срок не истёк; (2) статус «Активен». Инженер без заполненного `certificate_valid_until` блокировке по сроку не подлежит. Подробнее — в UC-901.
- История продления сертификатов не ведётся: одна актуальная запись на пару (инженер, модель). Все изменения фиксируются в аудит-логе.
- Справочник моделей оборудования ведётся отдельно (Модуль 10). Если модель удалена из справочника — компетенции по ней помечаются как устаревшие, но не удаляются.
