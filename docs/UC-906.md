# UC-906: Добавить комментарий к заявке

**Версия:** 1.2 | **Дата:** 31.03.2026 | **Статус:** На проверке

> **v1.1:** добавлена возможность клиенту добавлять комментарии через веб-портал / мобильное приложение; уточнена видимость комментариев для клиента.
> **v1.2:** добавлен поток удаления комментария (soft delete) — П-1 аудита документации.

## Основной поток

1. Пользователь открывает вкладку **«Комментарии»** в карточке заявки.
2. Система отображает хронологическую ленту существующих комментариев с разметкой:
   - Внутренние — серый фон, метка «Внутренний», видны только сотрудникам;
   - Внешние от сотрудников — белый фон, иконка «Видно клиенту», автор и дата/время;
   - Комментарии клиента — голубой фон, метка «От клиента», автор (имя контактного лица) и дата/время.
3. Пользователь нажимает **«+ Добавить комментарий»**.
4. Система отображает форму: текстовое поле + переключатель **Внутренний / Внешний**.
5. По умолчанию тип = **Внутренний**.
6. Пользователь вводит текст и выбирает тип.
7. Пользователь нажимает **«Отправить»**.
8. Система сохраняет комментарий с: текст, тип, `author_type = employee`, автор, timestamp.
9. Если тип **Внешний** — комментарий будет включён в следующее email-уведомление клиенту (при смене статуса → BR-F-910).
10. Система обновляет ленту комментариев.

---

## Альтернативные потоки

**АП-1: Редактирование своего комментария**
Пользователь может редактировать собственный комментарий в течение 10 минут после создания.
После 10 минут — редактирование недоступно.
При редактировании рядом с комментарием появляется метка «Изменён [время]».

**АП-3: Удаление комментария (только Администратор)**

1. Администратор нажимает **«Удалить»** у нужного комментария (кнопка видна только администратору).
2. Система показывает подтверждение с обязательным полем **«Причина удаления»**.
3. Администратор вводит причину и подтверждает.
4. Система применяет soft delete (`is_deleted = true`), фиксирует в аудит-логе: `action: delete`, `entity: comment`, причина, автор удаления, timestamp.
5. Комментарий скрывается из ленты для всех пользователей. В аудит-логе запись остаётся.

> **Примечание:** авторы (инженеры, менеджеры) могут **редактировать** свой комментарий в течение 10 минут (АП-1), но не удалять. Удаление — исключительное право Администратора.

**АП-2: Добавление комментария клиентом (веб-портал / мобильное приложение)**

1. Авторизованный клиент открывает карточку своей заявки на портале или в приложении.
2. Система отображает ленту комментариев: только **внешние** комментарии сотрудников и **все собственные** комментарии клиента — с датой и временем каждого.
3. Клиент нажимает **«Добавить комментарий»** (переключатель Внутренний/Внешний отсутствует — все комментарии клиента всегда внешние).
4. Клиент вводит текст и нажимает **«Отправить»**.
5. Система сохраняет комментарий: `type = external`, `author_type = client`, `author_client_id` = ID контакта клиента, timestamp.
6. Комментарий немедленно отображается в ленте клиента с именем контактного лица и датой/временем.
7. В карточке заявки на стороне сотрудников комментарий отображается с меткой «От клиента» (голубой фон) — сразу виден без перезагрузки.
8. Сотрудники получают уведомление о новом комментарии от клиента (Руководитель сервиса + назначенный инженер).

---

## Исключения

**ИП-1: Пустой комментарий**
→ Кнопка «Отправить» неактивна, пока текстовое поле пустое.

**ИП-2: Добавление комментария к закрытой заявке**
→ Форма добавления не отображается ни для сотрудников, ни для клиента. Существующие комментарии доступны только для чтения.

**ИП-3: Клиент пытается добавить комментарий к чужой заявке**
→ Система блокирует доступ: клиент видит только заявки своей организации. Карточка чужой заявки недоступна.

## Примечания / Бизнес-правила

- **BR-F-911:** два типа комментариев — `internal` (только для сотрудников) и `external` (видны клиенту). Клиентские комментарии всегда `external`.
- **BR-F-912:** клиент видит в ленте: все `external` комментарии сотрудников + все свои комментарии — с датой и временем каждого. Комментарии клиента отображаются у сотрудников с меткой «От клиента». При новом комментарии от клиента — уведомление Руководителю сервиса и назначенному инженеру.
- **BR-R-009:** физическое удаление комментариев запрещено (soft delete).
- Внешние комментарии отображаются на публичной странице статуса заявки (без авторизации, по уникальному токену → BR-F-910).
- Комментарии клиента с портала/приложения хранятся с `author_type = client` и `author_client_id`; `author_id` = NULL.
- Администратор может удалить любой комментарий (soft delete) с обязательным указанием причины.
