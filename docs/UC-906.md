# UC-906: Добавить комментарий к заявке

**Версия:** 1.0 | **Дата:** 26.03.2026 | **Статус:** На проверке

---

## Паспорт

| Поле | Значение |
|---|---|
| **ID** | UC-906 |
| **Модуль** | Модуль 9: Заявки на обслуживание |
| **Основной актор** | Сервисный инженер |
| **Второстепенные акторы** | Руководитель сервиса, Менеджер по продажам (все могут комментировать) |
| **Связанные BR-F** | BR-F-911 |
| **Связанные BR-R** | BR-R-009 (комментарии не удаляются физически) |
| **Приоритет** | Must |
| **Предусловия** | Заявка существует, статус — не «Закрыта»; пользователь авторизован |
| **Постусловия** | Комментарий сохранён; внешний комментарий включён в email клиенту при следующей смене статуса |
| **Триггер** | Пользователь открывает вкладку **«Комментарии»** в карточке заявки и нажимает **«+ Добавить»** |

---

## Основной поток

1. Пользователь открывает вкладку **«Комментарии»** в карточке заявки.
2. Система отображает хронологическую ленту существующих комментариев с разметкой: внутренние — серый фон, внешние — белый фон с иконкой «Видно клиенту».
3. Пользователь нажимает **«+ Добавить комментарий»**.
4. Система отображает форму: текстовое поле + переключатель **Внутренний / Внешний**.
5. По умолчанию тип = **Внутренний**.
6. Пользователь вводит текст и выбирает тип.
7. Пользователь нажимает **«Отправить»**.
8. Система сохраняет комментарий с: текст, тип, автор, timestamp.
9. Если тип **Внешний** — комментарий будет включён в следующее email-уведомление клиенту (при смене статуса → BR-F-910).
10. Система обновляет ленту комментариев.

---

## Альтернативные потоки

**АП-1: Редактирование своего комментария**
Пользователь может редактировать собственный комментарий в течение 10 минут после создания.
После 10 минут — редактирование недоступно.
При редактировании рядом с комментарием появляется метка «Изменён [время]».

---

## Исключения

**ИП-1: Пустой комментарий**
→ Кнопка «Отправить» неактивна, пока текстовое поле пустое.

**ИП-2: Добавление комментария к закрытой заявке**
→ Форма добавления не отображается. Существующие комментарии доступны только для чтения.

## Acceptance Criteria

```gherkin
Feature: Комментарии к заявке

  Scenario: Добавление внутреннего комментария
    Given заявка #42 в статусе «В работе»
    When инженер добавляет комментарий «Нужна замена ролика» с типом «Внутренний»
    Then комментарий сохранён с type=internal
    And клиент не получает уведомление об этом комментарии

  Scenario: Внешний комментарий включается в email клиенту
    Given заявка #42, добавлен внешний комментарий «Ожидаем поставку запчасти»
    When Руководитель сервиса меняет статус заявки на «Ожидание запчасти»
    Then email клиенту содержит текст «Ожидаем поставку запчасти»

  Scenario: Клиент видит только внешние комментарии
    Given заявка имеет 2 внутренних и 1 внешний комментарий
    When клиент открывает публичную страницу статуса заявки
    Then отображается только 1 внешний комментарий

  Scenario: Редактирование комментария в течение 10 минут
    Given комментарий добавлен 5 минут назад
    When автор редактирует текст и сохраняет
    Then комментарий обновлён с меткой «Изменён»

  Scenario: Редактирование недоступно после 10 минут
    Given комментарий добавлен 15 минут назад
    Then кнопка «Редактировать» не отображается
```

---

## Примечания / Бизнес-правила

- **BR-F-911:** два типа комментариев — internal (только для сотрудников) и external (видны клиенту).
- **BR-R-009:** физическое удаление комментариев запрещено (soft delete).
- Внешние комментарии отображаются на публичной странице статуса заявки (без авторизации, по уникальному токену → BR-F-910).
- Администратор может удалить любой комментарий (soft delete) с обязательным указанием причины.
