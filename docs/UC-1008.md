# UC-1008: Передача оборудования клиенту

**Версия:** 1.1 | **Дата:** 31.03.2026 | **Статус:** Рабочий

> **v1.1:** устранено противоречие П-3 аудита: вместо новых таблиц используются существующие `equipment_history` и `equipment_documents`.

---

## Паспорт

| Поле | Значение |
|---|---|
| **ID** | UC-1008 |
| **Модуль** | Модуль 10: Учёт оборудования |
| **Основной актор** | Руководитель сервиса |
| **Второстепенные акторы** | Система (фиксация истории передач) |
| **Связанные BR-F** | BR-F-1008, BR-F-1012 |
| **Связанные BR-R** | BR-R-009 (история передач не удаляется), BR-R-011 (допустимые типы файлов) |
| **Приоритет** | Must |
| **Предусловия** | Паспорт оборудования существует; поле `client_id` = NULL (оборудование не закреплено за клиентом); пользователь авторизован с ролью **Руководитель сервиса** или **Администратор** |
| **Постусловия** | Оборудование закреплено за клиентом; в истории передач появляется новая запись; прикреплённые документы сохранены |
| **Триггер** | Пользователь открывает паспорт оборудования без клиента → нажимает **«Передать клиенту»** |

---

## Основной поток

1. Пользователь открывает паспорт оборудования (статус = «Активно», без клиента).
2. Нажимает **«Передать клиенту»**.
3. Система отображает форму передачи.
4. Пользователь заполняет обязательные поля:
   - Клиент-получатель (выбор из справочника)
   - Дата передачи (по умолчанию — сегодня, редактируемая)
5. Опционально:
   - Адрес установки (куда устанавливается оборудование)
   - Номер акта передачи
   - Примечания
   - Вложения — скан акта передачи или иные документы (BR-F-1012)
6. Пользователь нажимает **«Сохранить»**.
7. Система валидирует обязательные поля.
8. Система выполняет:
   - Устанавливает `client_id` = выбранный клиент
   - Если указан адрес установки — обновляет `install_address` оборудования
   - Создаёт запись в таблице `equipment_transfer_history`:
     `from_client_id = NULL`, `to_client_id`, `transfer_date`, `transfer_type = initial_delivery`, `act_number`, `notes`, `recorded_by`, `recorded_at`
   - Если загружены файлы — сохраняет их и привязывает к записи истории (BR-F-1012)
9. Система отображает сообщение: *«Оборудование передано клиенту [Название]»*.
10. Паспорт обновляется: отображается клиент, адрес, дата передачи.

---

## Альтернативные потоки

**АП-1: Нет нужного клиента в справочнике**
Рядом с полем «Клиент» отображается ссылка *«+ Добавить клиента»* — открывает форму создания клиента в новой вкладке. После создания пользователь возвращается, обновляет список и выбирает нового клиента.

**АП-2: Файл не прикреплён**
Прикрепление файлов не является обязательным. Передача выполняется без документов.

**АП-3: Адрес установки не указан**
Поле `install_address` не обновляется. Адрес можно заполнить позднее через редактирование паспорта (UC-1001).

---

## Исключения

**ИП-1: Клиент не выбран**
→ Кнопка «Сохранить» неактивна. Подсказка: *«Выберите клиента-получателя»*.

**ИП-2: Дата передачи не заполнена**
→ Кнопка «Сохранить» неактивна. Подсказка: *«Укажите дату передачи»*.

**ИП-3: Оборудование в статусе «Списано»**
→ Кнопка «Передать клиенту» недоступна. Подсказка: *«Нельзя передать списанное оборудование»*.

**ИП-4: Оборудование в статусе «На ремонте»**
→ Система предупреждает: *«Оборудование находится в ремонте (заявка #[N]). Передача возможна, но рекомендуется закрыть заявку сначала»*. Пользователь может продолжить или отменить.

**ИП-5: Недопустимый формат файла (BR-R-011)**
→ Система отклоняет файл: *«Недопустимый формат. Допустимые форматы: PDF, JPG, PNG, DOCX»*. Остальные файлы и форма не затрагиваются.

**ИП-6: Размер файла превышает лимит**
→ Система отклоняет файл: *«Файл [имя] превышает максимальный размер (10 МБ)»*.

## Ключевые данные

### Форма передачи

| Поле | Обязательное | Тип | Описание |
|---|---|---|---|
| client_id | Да | FK → clients | Клиент-получатель |
| transfer_date | Да | date | Дата передачи; по умолчанию = сегодня |
| install_address | Нет | text | Адрес установки (обновляет паспорт) |
| act_number | Нет | varchar(64) | Номер акта приёма-передачи |
| notes | Нет | text | Примечания |
| files | Нет | file[] | Скан акта и иные документы |

### Запись в существующей таблице `equipment_history`

Используется существующая таблица ER (BR-R-009). Новых таблиц не создаётся.

| Поле в `equipment_history` | Тип | Значение при первичной передаче |
|---|---|---|
| equipment_id | FK → equipment.id, NN | Единица оборудования |
| event_type | enum | `initial_assignment` |
| from_client_id | FK → clients.id, NULL | NULL (нет прежнего клиента) |
| to_client_id | FK → clients.id, NULL | Выбранный клиент-получатель |
| event_date | date | Дата передачи (поле переименовано из `return_date` — см. ER v1.1) |
| act_number | varchar(64) | Номер акта (поле добавлено в ER v1.1) |
| reason | text | Примечания |
| recorded_by | FK → users.id, NN | Кто оформил |
| recorded_at | timestamp, NN | Дата записи |

### Вложения в существующей таблице `equipment_documents`

Файлы хранятся в существующей таблице `equipment_documents` (BR-F-1004) с `doc_type = 'act'`.
Добавлена связь с историей передач через новое поле `history_id` (см. ER v1.1).

| Поле в `equipment_documents` | Значение |
|---|---|
| equipment_id | FK → equipment.id |
| history_id | FK → equipment_history.id (новое поле, nullable) |
| doc_type | `act` |
| file_name | Оригинальное имя файла |
| file_url | Путь в хранилище |
| file_size_kb | Размер в КБ |
| mime_type | MIME-тип |
| uploaded_by | FK → users.id |
| uploaded_at | timestamp |

## Бизнес-правила

**BR-F-1008 — Логика первичной передачи оборудования клиенту**
Передача доступна только когда `client_id = NULL`. При сохранении:
- `client_id` обновляется до выбранного клиента
- `status` остаётся `active` (не меняется)
- `transfer_type = initial_delivery` фиксируется в истории

**BR-F-1012 — Прикрепление документов к акту передачи**
- Допустимые форматы: PDF, JPG, JPEG, PNG, DOCX (BR-R-011)
- Максимальный размер одного файла: 10 МБ
- Максимальное количество файлов за одну передачу: 10
- Файлы хранятся постоянно, не удаляются при изменении статуса оборудования
- Доступны для скачивания из вкладки «История передач» паспорта

**BR-R-009 — Записи истории не удаляются**
История передач ведётся с момента первой привязки. Записи доступны для просмотра на вкладке «История передач» паспорта оборудования.

**BR-R-011 — Допустимые форматы вложений**
PDF, JPG, JPEG, PNG, DOCX. Максимальный размер: 10 МБ на файл.

## Связи с другими UC

| UC | Связь |
|---|---|
| UC-1001 | Паспорт оборудования; поле `install_address` обновляется при передаче |
| UC-1003 | Передача оборудования другому клиенту (когда клиент уже есть, `transfer_type = re_transfer`) |
| UC-1007 | Возврат оборудования от клиента (`transfer_type = return_from_client`) |

> UC-1003 (re-transfer) и UC-1007 (return) используют ту же таблицу `equipment_transfer_history` с разными `transfer_type`. UC-1008 является начальной точкой жизненного цикла передачи.

## Примечания

- UC-1008 закрывает пробел между созданием паспорта (UC-1001) и передачей существующего оборудования другому клиенту (UC-1003): описывает **первичную** привязку оборудования к клиенту, когда оно ещё ни за кем не числится.
- Таблица `equipment_history` унифицирует всю историю движения оборудования: `event_type = initial_assignment` (UC-1008), `transfer` (UC-1003), `return` (UC-1007), `write_off` (UC-1003).
- Файлы-вложения хранятся в `equipment_documents` с `doc_type = 'act'` и привязкой к записи истории через `history_id`.
- В ER v1.1: поле `return_date` переименовано в `event_date`; добавлены поля `act_number` и `history_id` (в `equipment_documents`).
