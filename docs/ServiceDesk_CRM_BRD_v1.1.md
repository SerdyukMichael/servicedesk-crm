## 1. Резюме проекта

### 1.1 Цель и контекст

Компания является дистрибьютором и сервисной организацией по продаже и обслуживанию банкоматов и карт-машин (производитель -- Matica). Бизнес включает полный цикл: продажу оборудования клиентам (банки, финансовые организации, ритейл), гарантийное и постгарантийное обслуживание, поставку запчастей и расходных материалов, а также закупку оборудования и комплектующих у вендоров.

Цель проекта -- создание единой системы автоматизации (ServiceDesk CRM), которая объединит все аспекты взаимодействия с клиентами и вендорами: от первого контакта и коммерческого предложения до обслуживания проданного оборудования и управления складом запчастей.

### 1.2 Бизнес-цели (SMART)

|---|---|---|---|---|

| BG-005 | Обеспечить прозрачность складских остатков | Расхождение учетных и фактических остатков | <= 2% | 6 мес. после запуска |

### 1.3 Границы проекта

**В рамках проекта (In Scope):**

- CRM: ведение клиентов, контактов, истории взаимодействий
- Сделки: полный цикл продажи оборудования, запчастей и услуг
- Прайс-листы: ведение и актуализация цен на запчасти, расходники, услуги
- Договоры обслуживания: учет контрактов, условий SLA, сроков действия
- Выставление счетов и контроль оплаты
- Документооборот: счета, акты, накладные, договоры
- Заявки на обслуживание: прием, назначение, контроль SLA
- Учет оборудования: паспорта устройств, история ремонтов, возвраты
- Склад: приход, расход, инвентаризация, минимальные остатки
- Вендоры: закупка оборудования и запчастей, контроль поставок, рекламации
- Закупки: полный цикл от потребности до приемки
- Учет персонала: инженеры, менеджеры, компетенции
- Отчеты и аналитика: дашборд руководителя, отчеты по модулям
- Уведомления: единая система оповещений (email, push, in-app)

**За рамками проекта (Out of Scope):**

- Бухгалтерский учет (интеграция с 1С / внешней бухгалтерией)
- Расчет заработной платы
- Управление проектами (Gantt, Agile-доски)
- Электронный документооборот (ЭДО) -- только подготовка документов
- GPS-трекинг инженеров

## 3. Функциональные требования

**Терминология (см. также Глоссарий в разделе 7):**
- **Сделка (Deal)** -- коммерческая операция в модуле CRM: путь клиента от первого контакта до закрытия (воронка продаж). Одна сделка может порождать один или несколько заказов на продажу.
- **Заказ на продажу (Sales Order)** -- документ оформления конкретной продажи товаров/услуг в рамках сделки (модуль 3). Содержит спецификацию, цены, скидки.
- **Заявка на обслуживание (Service Request)** -- обращение клиента на ремонт или ТО оборудования (модуль 9).

**Общие правила прикрепления файлов** (применяются ко всем модулям, где требуется прикрепление файлов: BR-F-209, BR-F-604, BR-F-915, BR-F-1004):
- Допустимые форматы: PDF, JPEG, PNG, DOCX, XLSX, ZIP
- Максимальный размер одного файла: 20 МБ
- Максимальное количество файлов на одну сущность: 50

## 7. Глоссарий

| Термин | Определение |
|---|---|
| Банкомат (ATM) | Автоматическое устройство для выдачи наличных и проведения банковских операций |
| Карт-машина | Устройство для персонализации и выпуска пластиковых карт (производитель Matica) |
| Вендор | Производитель или поставщик оборудования и комплектующих (в контексте проекта -- Matica и другие возможные поставщики) |
| Сделка (Deal) | Коммерческая операция в модуле CRM: путь клиента от первого контакта до закрытия (воронка продаж) |
| Заказ на продажу (Sales Order) | Документ оформления конкретной продажи товаров/услуг в рамках сделки. Содержит спецификацию, цены, скидки |
| Заявка на обслуживание (Service Request) | Обращение клиента на ремонт, ТО или установку оборудования |
| SLA | Service Level Agreement -- соглашение об уровне обслуживания, определяющее время реакции и время решения |
| Договор обслуживания | Контракт с клиентом, определяющий тип SLA (Премиум / Стандарт), покрытое оборудование, условия и стоимость |
| ТО | Техническое обслуживание -- плановые профилактические работы по графику |
| КП | Коммерческое предложение -- документ с предложением товаров/услуг и их стоимости |
| Faktura VAT | Счет-фактура по польскому законодательству с налогом на добавленную стоимость |
| Faktura proforma | Предварительный счет, не являющийся налоговым документом |
| Faktura korygujaca | Корректировочная фактура для исправления ранее выставленного документа |
| Faktura zakupu | Входящий счет от поставщика |
| JPK | Jednolity Plik Kontrolny -- единый контрольный файл для польской налоговой (XML) |
| JPK_FA | Структура JPK для отчетности по фактурам |
| JPK_MAG | Структура JPK для складской отчетности |
| RODO | Rozporzadzenie o Ochronie Danych Osobowych -- польская реализация GDPR |
| NIP | Numer Identyfikacji Podatkowej -- польский налоговый идентификатор (аналог ИНН) |
| WZ | Wydanie Zewnetrzne -- накладная на внешнюю отгрузку товара со склада |
| PZ | Przyjecie Zewnetrzne -- документ приемки товара на склад от внешнего поставщика |
| FIFO | First In First Out -- метод учета, при котором первые поступившие товары расходуются первыми |
| Soft delete | Механизм "мягкого удаления" -- запись помечается как удаленная, но физически не стирается из БД |
| RBAC | Role-Based Access Control -- управление доступом к функциям на основе ролей пользователей |
