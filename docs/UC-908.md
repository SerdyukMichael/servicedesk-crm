## Основной поток — Создание графика ТО

1. Руководитель сервиса открывает паспорт оборудования → вкладка **«График ТО»**.
2. Система отображает текущий график (пустой или существующий).
3. Руководитель сервиса нажимает **«Настроить график»**.
4. Выбирает **периодичность**: ежемесячно / ежеквартально / раз в полгода / ежегодно.
5. Указывает **дату первого ТО** (отправную точку для расчёта последующих).
6. Нажимает **«Сохранить»**.
7. Система рассчитывает и отображает список плановых дат ТО на ближайший год.

---

## Основной поток — Автоматические действия системы

- За **14 дней** до даты ТО: система отправляет уведомление Руководителю сервиса (email + in-app) (→ BR-F-1009).
- За **7 дней** до даты ТО: система автоматически создаёт заявку типа «Плановое ТО» (→ UC-901, АП авто-создания).
- После создания заявки соответствующая дата в графике помечается: ✅ «Заявка создана #[N]».

---

## Исключения

**ИП-1: Дата первого ТО в прошлом**
→ Система предупреждает: *«Указанная дата в прошлом. Первая заявка будет создана сразу»*.

## Примечания / Бизнес-правила

- **BR-F-909:** авто-создание заявки за 7 дней — системный процесс, не требует участия пользователя.
- Если заявка по плановому ТО уже создана (например, вручную), повторное авто-создание не происходит.
- График ТО привязан к конкретной единице оборудования, не к модели.
