# UC-909: Поиск и фильтрация заявок

**Версия:** 1.0 | **Дата:** 26.03.2026 | **Статус:** На проверке

---

## Паспорт

| Поле | Значение |
|---|---|
| **ID** | UC-909 |
| **Модуль** | Модуль 9: Заявки на обслуживание |
| **Основной актор** | Руководитель сервиса |
| **Второстепенные акторы** | Сервисный инженер (видит только свои заявки), Директор (все заявки, только чтение) |
| **Связанные BR-F** | BR-F-912 |
| **Связанные BR-R** | — |
| **Приоритет** | Must |
| **Предусловия** | Пользователь авторизован |
| **Постусловия** | Нет изменений — только чтение. Переход в карточку заявки по клику |
| **Триггер** | Пользователь открывает раздел **«Заявки»** |

---

## Основной поток

1. Пользователь открывает раздел **«Заявки»**.
2. Система отображает список заявок (с учётом прав роли: инженер видит только свои).
3. Пользователь применяет фильтры и/или вводит поисковый запрос.
4. Система отображает отфильтрованный список.
5. Пользователь кликает на заявку — открывается карточка.

---

## Доступные фильтры (BR-F-912)

| Фильтр | Тип | Примечание |
|---|---|---|
| Статус | Мультивыбор | Новая, Назначена, В работе, Ожидание запчасти, Выполнена, На проверке, Закрыта |
| Клиент | Поиск + выбор | Из справочника клиентов |
| Назначенный инженер | Выбор | Из списка инженеров |
| Дата создания | Диапазон дат | От — До |
| Приоритет | Мультивыбор | Критичный, Высокий, Средний, Низкий |
| Серийный номер | Текст | Точное и частичное совпадение |
| Тип работ | Мультивыбор | Гарантийный ремонт, Плановое ТО, Внеплановый ремонт, Установка |
| Нарушение SLA | Чекбокс | Только заявки с sla_*_violated = true |

## Ключевые данные (параметры фильтрации)

| Параметр | Тип | Значения |
| --- | --- | --- |
| status | enum (мультивыбор) | Новая, Назначена, В работе, Ожидание запчасти, Выполнена, На проверке, Закрыта |
| client_id | FK → clients | Выбор из справочника |
| assigned_engineer_id | FK → users | Выбор из списка инженеров |
| created_at | daterange | От — До |
| priority | enum (мультивыбор) | Критичный, Высокий, Средний, Низкий |
| serial_number | varchar | Частичное совпадение |

| sla_violated | boolean | Только просроченные |

---

## Исключения

**ИП-1: Поиск не дал результатов**
→ Система отображает: *«Заявки не найдены. Измените параметры фильтра»*. Кнопка «Сбросить» доступна.

**ИП-2: Инженер пытается увидеть чужие заявки через фильтр**
→ Фильтр «Назначенный инженер» у роли Инженер скрыт — список автоматически ограничен его заявками.

## Acceptance Criteria

```gherkin
Feature: Поиск и фильтрация заявок

  Scenario: Фильтрация по статусу и клиенту
    Given в системе 50 заявок разных клиентов и статусов
    When пользователь выбирает статус «В работе» и клиента «Банк Альфа»
    Then отображаются только заявки Банк Альфа в статусе «В работе»

  Scenario: Инженер видит только свои заявки
    Given в системе 20 заявок, 5 из них назначены на инженера «Петров»
    When «Петров» открывает раздел «Заявки»
    Then отображается 5 заявок

  Scenario: Поиск по серийному номеру (частичное совпадение)
    When пользователь вводит «SN-12» в поле серийного номера
    Then отображаются все заявки, где серийный номер содержит «SN-12»
```

## Примечания / Бизнес-правила

- Инженер видит только заявки, назначенные на него (RBAC).
- Руководитель сервиса и Администратор видят все заявки.
- Директор видит все заявки, но без возможности изменения.
- Фильтры сохраняются в сессии (при переходе на другую страницу и возврате — фильтры активны).
