# UC-401: Создать счёт на оплату

**Версия:** 1.0 | **Дата:** 07.04.2026 | **Статус:** В работе

---

## Паспорт

| Поле | Значение |
|---|---|
| **ID** | UC-401 |
| **Модуль** | Модуль 4: Выставление счетов |
| **Основной актор** | Менеджер / Руководитель сервиса |
| **Второстепенные акторы** | Система (нумерация, расчёт суммы) |
| **Связанные BR-F** | BR-F-400, BR-F-403, BR-F-404, BR-F-407, BR-F-411 |
| **Связанные BR-R** | BR-P-002, BR-P-003 |
| **Приоритет** | Must |
| **Предусловия** | Пользователь авторизован с ролью `admin`, `svc_mgr` или `manager`; клиент существует в системе |
| **Постусловия** | Счёт создан в статусе «Черновик»; позиции зафиксированы |
| **Триггер** | Пользователь открывает «Счета → Создать счёт» или нажимает «Создать счёт» из карточки заявки / акта |

---

## Основной поток

1. Пользователь открывает форму создания счёта.
2. Система автоматически генерирует номер счёта по шаблону `INV/YYYY/MM/NNN`.
3. Пользователь выбирает **клиента** из справочника.
4. Пользователь (опционально) привязывает **заявку** (`ticket_id`).
5. Пользователь устанавливает **дату выставления** (по умолчанию — сегодня) и **срок оплаты**.
6. Пользователь добавляет **позиции счёта**:

   а. Нажимает «Добавить строку».

   б. Выбирает тип: **Услуга** (из ServiceCatalog) или **Запчасть** (из SparePart) или **Ручной ввод**.

   в. При выборе из справочника система автоматически подставляет: наименование, единицу, цену (BR-F-111).

   г. Пользователь указывает количество. Система вычисляет `total = qty × unit_price`.

   д. Пользователь вправе изменить цену вручную.

7. Пользователь устанавливает **ставку НДС** (0% / 10% / 20%).
8. Система автоматически вычисляет `subtotal`, `vat_amount`, `total_amount`.
9. Пользователь добавляет **примечания** (необязательно).
10. Пользователь нажимает **«Сохранить черновик»**.
11. Система сохраняет счёт в статусе `draft`. Цены фиксируются (BR-P-003).

---

## Альтернативные потоки

**АП-1: Создание счёта на основе акта (→ UC-403)**
На шаге 1 пользователь выбирает «Создать из акта».
Система копирует позиции акта в строки счёта (→ UC-403).

**АП-2: Счёт без позиций**
Пользователь пытается сохранить счёт без строк.
→ Система предупреждает, но позволяет сохранить черновик (позиции можно добавить позже).

**АП-3: Гарантийный ремонт**
Если связанная заявка помечена как гарантийная (`is_warranty = true`), система отображает предупреждение: «Заявка гарантийная. Счёт клиенту не выставляется (BR-P-005)».
Пользователь может продолжить (если счёт нулевой — например, для отчётности) или отменить.

---

## Исключения

**ИП-1: Клиент не выбран**
→ Система блокирует: «Выберите клиента».

**ИП-2: Количество ≤ 0**
→ Система блокирует: «Количество должно быть больше 0».

**ИП-3: Цена отрицательная**
→ Система блокирует: «Цена не может быть отрицательной».

## Ключевые данные

| Поле | Обязательное | Тип | Примечание |
|---|---|---|---|
| number | Авто | varchar(32) | INV/YYYY/MM/NNN |
| client_id | Да | FK → clients | — |
| ticket_id | Нет | FK → tickets | Опциональная привязка |
| issue_date | Да | date | По умолчанию сегодня |
| due_date | Нет | date | — |
| status | Авто | enum | draft при создании |
| vat_rate | Да | decimal | 0 / 10 / 20 |
| items[].item_type | Да | enum | service / part / manual |
| items[].service_id | Нет | FK → service_catalog | При item_type=service |
| items[].part_id | Нет | FK → spare_parts | При item_type=part |
| items[].name | Да | varchar(512) | Копируется из справочника |
| items[].quantity | Да | decimal | > 0 |
| items[].unit_price | Да | decimal | Из прайса, редактируемая |
| items[].total | Авто | decimal | qty × unit_price |

## Acceptance Criteria

```gherkin
Feature: Создание счёта

  Scenario: Создание счёта с услугой из справочника
    Given существует услуга SRV-001 «Диагностика» по цене 1500 руб.
    When пользователь создаёт счёт для клиента «Банк Альфа», добавляет SRV-001 × 2 шт.
    Then строка счёта содержит: name=«Диагностика», unit_price=1500, total=3000
    And счёт сохранён в статусе «Черновик»

  Scenario: Цена в строке редактируется
    Given SRV-001 стоит 1500 руб.
    When пользователь меняет цену в строке на 1000 руб.
    Then total = 1000 × qty
    And в прайс-листе SRV-001 остаётся 1500 руб.

  Scenario: Изменение прайса не влияет на сохранённый счёт
    Given счёт #5 создан с SRV-001 по 1500 руб.
    When администратор меняет цену SRV-001 на 2000 руб.
    Then счёт #5 по-прежнему содержит unit_price=1500
```
