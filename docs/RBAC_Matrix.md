**Версия:** 2.3 | **Дата:** 08.04.2026 | **Статус:** Актуально

> Определяет, какие действия разрешены каждой роли.

## Роли системы

| Код | Роль | Краткое описание |
| --- | --- | --- |
| `admin` | Администратор | Управление пользователями, ролями, конфигурацией системы |
| `sales_mgr` | Менеджер по продажам | CRM-клиенты, договоры, продажи |
| `svc_mgr` | Руководитель сервиса | Полный доступ к заявкам, оборудованию, команде |
| `engineer` | Сервисный инженер | Собственные заявки: работа с актами, статусами, комментариями |
| `manager` | Менеджер | Создание и просмотр заявок, работа с клиентами |
| `warehouse` | Кладовщик | Склад запчастей |
| `accountant` | Бухгалтер | Счета, акты, финансовые документы |
| `director` | Директор | Только чтение по всем модулям + отчёты |
| `client_user` | Пользователь клиента | Доступ к данным своей организации через встроенный портал (UC-301) |

## Роль `client_user` — Пользователь клиента (портал)

Роль создаётся автоматически при выдаче доступа к порталу контактному лицу (UC-301).
Все данные жёстко ограничены **организацией контакта** (`users.client_id`). Фильтрация принудительная на уровне backend.

| Модуль | Действие | Доступ |
| --- | --- | --- |
| **Заявки** | Просмотреть список | 🔐 только своей организации |
| **Заявки** | Создать заявку | ✅ только для своей организации |
| **Заявки** | Просмотреть карточку | 🔐 только своей организации |
| **Заявки — комментарии** | Просмотреть комментарии | 🔐 только внешние (`is_internal=false`) своей организации |
| **Заявки — комментарии** | Добавить комментарий | ✅ всегда внешний (`is_internal` принудительно `false`) |
| **Заявки — комментарии** | Элемент «Внутренний комментарий» | — (скрыт на фронтенде) |
| **Заявки — акт** | Подписать акт выполненных работ | ✅ только для заявок своей организации |
| **Клиенты** | Просмотреть список | 🔐 только своя организация (1 запись) |
| **Клиенты** | Просмотреть карточку | 🔐 только своя организация |
| **Оборудование** | Просмотреть список / паспорт | 🔐 только своей организации |
| **Модели оборудования** | Просмотреть справочник | 📖 все |
| **Склад запчастей** | Любые действия | — (403 Forbidden) |
| **Счета** | Просмотреть список / карточку | 🔐 только своей организации |
| **Пользователи** | Просмотреть список | 🔐 только пользователи своей организации |
| **Пользователи** | Создание / редактирование | — |
| **Уведомления** | Свои уведомления | ✅ |

> **Технически:** при каждом запросе backend извлекает `current_user.client_id` и добавляет фильтр `WHERE client_id = ?`. Для склада запчастей — 403 Forbidden.
> Для комментариев — дополнительный фильтр `WHERE is_internal = false`. Подпись акта дополнительно проверяет `ticket.client_id == current_user.client_id`.
