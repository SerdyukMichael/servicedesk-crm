---

## Типы возврата

| Тип | Описание | Дальнейшие действия |
|---|---|---|
| Гарантийная замена | Дефект в гарантийный период | Система предлагает создать рекламацию вендору (→ BR-F-1111) |
| Выкуп б/у | Клиент продаёт оборудование обратно | — |

---

## Основной поток

1. Руководитель сервиса открывает паспорт → нажимает **«Оформить возврат»**.
2. Выбирает тип возврата.
3. Указывает дату возврата и причину.
4. Подтверждает.
5. Система:
   - Отвязывает оборудование от клиента (сохраняет историю: дата, клиент, тип возврата)
   - Меняет статус на **«Активно»** (без клиента) или **«Списано»** (при выкупе б/у, если требуется)
6. Если тип = «Гарантийная замена»: система предлагает *«Создать рекламацию вендору?»*. При согласии → открывается форма рекламации (→ Модуль 11).

---

## Альтернативные потоки

**АП-1: Отказ от создания рекламации**
При гарантийной замене система предлагает создать рекламацию. Пользователь нажимает «Пропустить» — рекламация не создаётся, возврат оформляется.

**АП-2: Возврат типа «Выкуп б/у»**
Система не предлагает рекламацию. Пользователь выбирает итоговый статус оборудования: «Активно» (без клиента) или «Списано».

## Исключения

**ИП-1: Возврат оборудования в статусе «Списано»**
→ Кнопка «Оформить возврат» недоступна. Подсказка: *«Нельзя оформить возврат списанного оборудования»*.

**ИП-2: Возврат оборудования с открытой заявкой**
→ Система предупреждает: *«У оборудования есть открытая заявка #[N]. Уверены?»*. Пользователь может продолжить или отменить.

**ИП-3: Причина возврата не указана**
→ Кнопка «Подтвердить» неактивна до заполнения поля «Причина».

## Acceptance Criteria

```gherkin
Feature: Возврат оборудования

  Scenario: Гарантийный возврат с рекламацией
    Given оборудование SN-12345 «На гарантии» у клиента «Банк Альфа»
    When оформляется возврат типа «Гарантийная замена»
    Then оборудование отвязывается от «Банк Альфа»
    And система предлагает создать рекламацию вендору

  Scenario: История возвратов сохраняется
    Given возврат оформлен
    Then на вкладке «История передач» паспорта есть запись о возврате

  Scenario: Нельзя вернуть списанное оборудование
    Given оборудование SN-99999 в статусе «Списано»
    When пользователь открывает паспорт
    Then кнопка «Оформить возврат» недоступна

  Scenario: Причина возврата обязательна
    When пользователь пытается подтвердить возврат без указания причины
    Then кнопка «Подтвердить» неактивна
```

---

## Примечания

- История всех передач и возвратов хранится на вкладке «История передач» паспорта. Записи не удаляются (BR-R-009).
- **Модуль 11 (Рекламации вендорам) — вне MVP.** При согласии пользователя система создаёт черновую запись рекламации (`vendor_claims`, статус `draft`) с данными: оборудование, серийный номер, тип дефекта из причины возврата. Полное оформление рекламации реализуется в Phase 3 (Модуль 11). Структура сущности `vendor_claims` описана в BRD (BR-F-1111).
