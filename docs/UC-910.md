# UC-910: Сформировать отчёт по заявкам

**Версия:** 1.0 | **Дата:** 26.03.2026 | **Статус:** На проверке

---

| Поле | Значение |
|---|---|
| **ID** | UC-910 |

---

## Основной поток

1. Пользователь открывает **«Отчёты» → «Отчёт по заявкам»**.
2. Система отображает параметры: период (диапазон дат), опциональные фильтры (инженер, клиент).
3. Пользователь задаёт период и нажимает **«Сформировать»**.
4. Система рассчитывает показатели и отображает отчёт.
5. Пользователь экспортирует в PDF или XLSX при необходимости.

---

## Состав отчёта (BR-F-908)

| Показатель | Описание |
|---|---|
| Количество заявок по статусам | Таблица: статус → количество |
| % соблюдения SLA реакции | (Заявки без нарушения / всего) × 100% |
| % соблюдения SLA решения | (Заявки без нарушения / всего) × 100% |
| Среднее время решения | В часах, по закрытым заявкам периода |
| Загрузка инженеров | Таблица: инженер → количество закрытых заявок |

## Acceptance Criteria

```gherkin
Feature: Отчёт по заявкам

  Scenario: Формирование отчёта за период
    Given за март 2026: 100 заявок, 90 без нарушения SLA реакции, среднее решение 18 ч.
    When Директор формирует отчёт за март 2026
    Then отображается: % SLA реакции = 90%, среднее время решения = 18 ч.

  Scenario: Экспорт в PDF
    Given отчёт сформирован
    When пользователь нажимает «Экспорт PDF»
    Then скачивается PDF-файл с данными отчёта
```

---

## Примечания / Бизнес-правила

- **BR-F-1501, BR-F-1502:** все отчёты поддерживают произвольный диапазон дат и экспорт в PDF и XLSX.
- Отчёт строится только по заявкам, созданным в выбранном периоде (не по закрытым в периоде).
