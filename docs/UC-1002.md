## Основной поток

1. Пользователь открывает паспорт оборудования → вкладка **«История ремонтов»**.
2. Система отображает хронологический список всех записей (новые сверху).
3. Пользователь раскрывает запись для просмотра деталей.
4. Пользователь кликает на номер заявки — переходит в карточку заявки.

---

## Состав записи истории ремонтов (BR-F-1001)

| Поле | Описание |
|---|---|
| Дата | Дата выполнения работ |
| Тип работ | Гарантийный ремонт / Плановое ТО / Внеплановый ремонт / Установка |
| Описание | Текст из акта выполненных работ |
| Инженер | ФИО назначенного инженера |
| Использованные запчасти | Список: артикул, наименование, количество |
| Номер заявки | Ссылка на заявку (→ UC-903) |

## Acceptance Criteria

```gherkin
Feature: История ремонтов оборудования

  Scenario: Отображение записей из закрытых заявок
    Given оборудование SN-12345 имеет 3 закрытые заявки
    When пользователь открывает вкладку «История ремонтов»
    Then отображаются 3 записи в хронологическом порядке

  Scenario: Переход в карточку заявки
    Given в истории есть запись с заявкой #SD-2026-000010
    When пользователь кликает на номер заявки
    Then открывается карточка заявки #SD-2026-000010
```

---

## Примечания / Бизнес-правила

- Записи истории создаются автоматически: при создании заявки — предварительная запись, при закрытии — финальная с деталями (BR-F-906).
- Записи нельзя добавлять или редактировать вручную — только через заявки.
- «ТОП-10 проблемных единиц» (по количеству ремонтов) — в отчёте по парку оборудования (→ UC-1007).
