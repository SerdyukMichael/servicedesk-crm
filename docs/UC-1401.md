# UC-1401: Настроить предпочтения уведомлений

**Версия:** 1.0 | **Дата:** 26.03.2026 | **Статус:** На проверке

---

## Паспорт

| Поле | Значение |
|---|---|
| **ID** | UC-1401 |
| **Модуль** | Модуль 14: Уведомления |
| **Основной актор** | Любой авторизованный пользователь |
| **Второстепенные акторы** | — |

---

## Основной поток

1. Пользователь открывает **«Профиль» → «Настройки уведомлений»**.
2. Система отображает матрицу: строки = типы событий, столбцы = каналы (Email / Push / In-app).
3. Для каждой ячейки — чекбокс включён/выключен.
4. Пользователь изменяет нужные настройки.
5. Нажимает **«Сохранить»**.
6. Система сохраняет предпочтения. Изменения вступают в силу немедленно.

---

## Типы событий и каналы (BR-F-1401)

| Событие | Email | Push | In-app |
|---|---|---|---|
| Новая заявка назначена мне | ✓ | ✓ | ✓ |
| Изменение статуса заявки | ✓ | ✓ | ✓ |
| Нарушение SLA (эскалация) | ✓ | — | ✓ |
| Приближение срока оплаты | ✓ | — | ✓ |
| Приближение планового ТО | ✓ | — | ✓ |
| Истечение гарантии | ✓ | — | ✓ |
| Новый комментарий к моей заявке | ✓ | ✓ | ✓ |

> Канал In-app нельзя отключить — системное требование.

---

## Исключения

**ИП-1: Попытка отключить in-app уведомления**
→ Чекбокс In-app заблокирован (disabled). Подсказка: *«In-app уведомления нельзя отключить»*.

---

## Acceptance Criteria

```gherkin
Feature: Настройки уведомлений

  Scenario: Отключение email для нового комментария
    Given пользователь «Петров» получает email при новом комментарии
    When он снимает чекбокс Email для события «Новый комментарий»
    Then следующий комментарий к его заявке не вызывает email
    And in-app уведомление всё равно отправляется

  Scenario: In-app нельзя отключить
    When пользователь пытается снять чекбокс In-app
    Then чекбокс остаётся активным (заблокирован)
```

---

## Примечания / Бизнес-правила

- **BR-F-1400:** три канала работают параллельно. Предпочтения пользователя влияют на Email и Push; In-app — всегда включён.
- Администратор может просматривать настройки уведомлений любого пользователя, но не изменять их за него.
