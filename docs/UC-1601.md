**Версия:** 1.0 | **Дата:** 19.04.2026 | **Статус:** В работе

## Основной поток: Изменение системной валюты

1. Администратор открывает раздел **«Настройки → Системная валюта»**.
2. Система отображает текущую настройку:
   - **Код валюты** (ISO 4217, 3 символа) — текущее значение
   - **Наименование валюты** — текущее значение
3. Администратор нажимает **«Изменить»**.
4. Система открывает форму редактирования с полями:
   - **Код валюты** — выбор из предустановленного списка или ввод вручную
   - **Наименование валюты** — текстовое поле
5. Администратор выбирает или вводит новые значения.
6. Администратор нажимает **«Сохранить»**.
7. Система валидирует: код — ровно 3 латинских символа (верхний регистр), наименование — не пустое.
8. Система сохраняет настройку.
9. Система отображает подтверждение: «Системная валюта обновлена. Новая валюта: [код] — [наименование]».
10. Все денежные суммы в интерфейсе отображаются с новым кодом/наименованием валюты.

---

## Альтернативные потоки

**АП-1: Выбор из предустановленного списка**
На шаге 4 администратор выбирает одно из предустановленных значений:

| Код | Наименование |
|---|---|
| RUB | Российский рубль |
| KZT | Казахстанский тенге |
| PLN | Польский злотый |

После выбора поля «Код» и «Наименование» заполняются автоматически. Администратор может скорректировать наименование вручную.

**АП-2: Ручной ввод произвольной валюты**
Администратор вводит произвольный ISO 4217 код и наименование (например, EUR / Евро).
Система принимает любой 3-символьный код при условии корректного формата.

**АП-3: Просмотр без изменений**
Администратор открывает раздел «Системная валюта» только для просмотра — не нажимает «Изменить».
Система отображает текущие значения в режиме чтения.

---

## Исключения

**ИП-1: Код валюты не 3 символа**
→ Система блокирует: «Код валюты должен содержать ровно 3 латинских символа в верхнем регистре (например: RUB, KZT, PLN)».

**ИП-2: Наименование не заполнено**
→ Система блокирует: «Введите наименование валюты».

**ИП-3: Попытка доступа без роли admin**
→ Раздел «Системные настройки» не отображается в навигации. При прямом переходе по URL — ошибка 403.

## Ключевые данные

### Настройка (таблица `system_settings`)

| Поле | Обязательное | Тип | Ограничения | Значение по умолчанию |
|---|---|---|---|---|
| key | Да | varchar(64) | `currency_code` — уникальный ключ | — |
| value | Да | varchar(255) | Для `currency_code`: 3 символа A-Z | `RUB` |

### Хранение двух параметров

| Ключ (`key`) | Описание | Пример значения |
|---|---|---|
| `currency_code` | ISO 4217 код валюты | `RUB` |
| `currency_name` | Наименование валюты | `Российский рубль` |

---

## Применение настройки в системе

Настройка системной валюты применяется во всех местах, где отображаются денежные суммы:

| Модуль | Где применяется |
|---|---|
| Прайс-лист услуг (UC-101) | Поле «Валюта» при создании/редактировании позиции |
| Прайс-лист матценностей (UC-102) | Поле «Валюта» при установке/изменении цены; история цен |
| Счета (UC-401) | Отображение сумм позиций, итого, НДС |
| Счёт из акта (UC-403) | Отображение сумм позиций при копировании из акта |
| Карточка заявки | Суммы в акте, диалог предупреждения при несовпадении сумм |
| Список счетов | Колонка «Сумма» |

## Acceptance Criteria

```gherkin
Feature: Настройка системной валюты

  Scenario: Успешное изменение валюты на KZT
    Given администратор открыл «Настройки → Системная валюта»
    And текущая валюта: код=RUB, наименование=«Российский рубль»
    When нажимает «Изменить» и выбирает из списка KZT / «Казахстанский тенге»
    And нажимает «Сохранить»
    Then система сохраняет: currency_code=KZT, currency_name=«Казахстанский тенге»
    And в списке счетов суммы отображаются с кодом KZT
    And в форме прайс-листа поле «Валюта» показывает KZT

  Scenario: Ввод произвольной валюты
    Given администратор открыл форму изменения валюты
    When вводит код=EUR, наименование=«Евро»
    And нажимает «Сохранить»
    Then система принимает EUR как корректный код
    And сохраняет currency_code=EUR, currency_name=«Евро»

  Scenario: Блокировка некорректного кода
    Given администратор открыл форму изменения валюты
    When вводит код=«RU» (2 символа)
    And нажимает «Сохранить»
    Then система показывает «Код валюты должен содержать ровно 3 латинских символа в верхнем регистре»
    And сохранение блокируется

  Scenario: Доступ запрещён для не-admin
    Given пользователь с ролью svc_mgr авторизован
    When переходит по URL /settings/currency
    Then система возвращает ошибку 403
    And раздел «Системные настройки» отсутствует в навигации

  Scenario: Значение по умолчанию при первой установке
    Given система установлена впервые
    When администратор открывает «Настройки → Системная валюта»
    Then отображается: код=RUB, наименование=«Российский рубль»
```
