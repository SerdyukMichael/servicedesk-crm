# UC-1403: Настроить SMTP-сервер для отправки email

**Версия:** 1.0 | **Дата:** 26.03.2026 | **Статус:** На проверке

---

## Паспорт

| Поле | Значение |
|---|---|
| **ID** | UC-1403 |
| **Модуль** | Модуль 14: Уведомления |
| **Основной актор** | Администратор |
| **Связанные BR-F** | BR-F-1404 |
| **Приоритет** | Must |
| **Предусловия** | Пользователь авторизован с ролью **Администратор**; доступ к внешнему SMTP-серверу имеется |
| **Постусловия** | SMTP-настройки сохранены; система использует их для отправки всех email-уведомлений |
| **Триггер** | Администратор открывает **«Настройки системы» → «Email / SMTP»** |

---

## Основной поток

1. Администратор открывает **«Настройки системы» → «Email / SMTP»**.
2. Система отображает форму с текущими настройками (или пустую при первом входе).
3. Администратор заполняет поля: SMTP-хост, порт, логин, пароль, TLS (вкл/выкл), адрес «От» (From), имя отправителя.
4. Нажимает **«Тестировать соединение»**.
5. Система пытается подключиться к SMTP-серверу и отправить тестовое письмо на email администратора.
6. Система отображает результат: ✅ «Соединение успешно, тестовое письмо отправлено» или ❌ «Ошибка: [описание]».
7. При успехе — Администратор нажимает **«Сохранить»**.
8. Система сохраняет настройки (пароль хранится в зашифрованном виде).

---

## Исключения

**ИП-1: Ошибка соединения**
→ Система отображает детали ошибки: *«Не удалось подключиться к [хост]:[порт]. Проверьте параметры и доступность сервера»*.
→ Настройки не сохраняются автоматически — только вручную после проверки.

**ИП-2: Сохранение без тестирования**
→ Система предупреждает: *«Рекомендуется протестировать соединение перед сохранением»*. Сохранение не блокируется.

## Ключевые данные

| Поле | Обязательное | Тип | Примечание |
|---|---|---|---|
| smtp_host | Да | varchar(255) | Хост SMTP-сервера |
| smtp_port | Да | integer | Обычно 25, 465, 587 |
| smtp_login | Да | varchar(255) | — |
| smtp_password | Да | varchar(255) | Хранится зашифрованным (AES-256) |
| tls_enabled | Да | boolean | — |
| from_address | Да | varchar(255) | Формат email |
| from_name | Нет | varchar(100) | Имя отправителя |

## Acceptance Criteria

```gherkin
Feature: Настройка SMTP

  Scenario: Успешная настройка и тест
    When Администратор вводит корректные параметры SMTP и нажимает «Тестировать»
    Then система показывает «Соединение успешно, тестовое письмо отправлено»
    And Администратор получает тестовое письмо на свой email

  Scenario: Ошибка соединения
    When Администратор вводит неверный хост и нажимает «Тестировать»
    Then система показывает ошибку с описанием проблемы
    And настройки не сохраняются автоматически

  Scenario: Пароль хранится зашифрованным
    Given SMTP-настройки сохранены
    When Администратор открывает страницу настроек повторно
    Then поле пароля отображается как «••••••••» (маскировано)
    And пароль не передаётся в открытом виде через API
```

---

## Примечания / Бизнес-правила

- **BR-F-1404:** SMTP-сервер настраивается администратором. Все email-уведомления системы используют единую конфигурацию.
- Пароль хранится зашифрованным (AES-256). В логах и API-ответах пароль никогда не передаётся в открытом виде.
- При смене SMTP-настроек все последующие письма используют новые параметры. Уже отправленные письма не затрагиваются.
