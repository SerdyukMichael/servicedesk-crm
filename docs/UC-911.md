# UC-911: Управлять шаблонами типовых работ

**Версия:** 1.0 | **Дата:** 26.03.2026 | **Статус:** На проверке

---

## Паспорт

| Поле | Значение |
|---|---|
| **ID** | UC-911 |
| **Модуль** | Модуль 9: Заявки на обслуживание |
| **Основной актор** | Руководитель сервиса |
| **Второстепенные акторы** | Администратор |

| **Связанные BR-R** | BR-R-009 (soft delete) |

| **Предусловия** | Пользователь авторизован с ролью **Руководитель сервиса** или **Администратор**; справочник моделей оборудования и номенклатура запчастей заполнены |
| **Постусловия** | Шаблон создан / изменён; доступен для применения при создании заявки (→ UC-901) |
| **Триггер** | Пользователь открывает **«Настройки» → «Шаблоны работ»** |

---

## Основной поток — Создание шаблона

1. Пользователь нажимает **«+ Создать шаблон»**.
2. Заполняет: название шаблона, модель оборудования (из справочника).
3. Добавляет позиции работ: описание работы (текст), последовательность.
4. Добавляет запчасти: выбор из номенклатуры, количество.
5. Нажимает **«Сохранить»**.
6. Шаблон появляется в списке и доступен при создании заявки.

---

## Исключения

**ИП-1: Шаблон без позиций работ**
→ Система блокирует: *«Добавьте хотя бы одну работу в шаблон»*.

---

## Ключевые данные

| Поле | Тип | Описание |
|---|---|---|
| name | varchar(200) | Название шаблона |
| equipment_model_id | FK → equipment_models | Привязка к модели |
| work_items | []text | Список работ |
| parts | []{ part_id, qty } | Список запчастей с количеством |

## Acceptance Criteria

```gherkin
Feature: Шаблоны типовых работ

  Scenario: Применение шаблона при создании заявки
    Given существует шаблон «ТО Matica XID 580i» с 3 работами и 2 запчастями
    When Руководитель сервиса создаёт заявку и выбирает этот шаблон
    Then форма предзаполняется 3 работами и 2 позициями запчастей

  Scenario: Блокировка сохранения пустого шаблона
    When пользователь создаёт шаблон без позиций работ
    Then система блокирует с ошибкой «Добавьте хотя бы одну работу»
```

---

## Примечания / Бизнес-правила

- Шаблон привязывается к модели оборудования. При создании заявки доступны шаблоны, соответствующие модели из заявки.
- Применение шаблона — необязательный шаг. Предзаполненные данные можно редактировать.
