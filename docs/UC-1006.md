# UC-1006: Сформировать отчёт по парку оборудования

**Версия:** 1.0 | **Дата:** 26.03.2026 | **Статус:** На проверке

---

## Паспорт

| Поле | Значение |
|---|---|
| **ID** | UC-1006 |
| **Модуль** | Модуль 10: Учёт оборудования |
| **Основной актор** | Директор |
| **Второстепенные акторы** | Руководитель сервиса |
| **Связанные BR-F** | BR-F-1008 |
| **Приоритет** | Should |
| **Предусловия** | Пользователь авторизован с ролью **Директор** или **Руководитель сервиса** |
| **Постусловия** | Отчёт сформирован; доступен для просмотра и экспорта |
| **Триггер** | **«Отчёты» → «Отчёт по парку оборудования»** |

---

## Основной поток

1. Пользователь открывает раздел отчётов → «Отчёт по парку оборудования».
2. Задаёт период анализа (диапазон дат).
3. Нажимает **«Сформировать»**.
4. Система формирует отчёт и отображает разделы.
5. Пользователь экспортирует в PDF или XLSX при необходимости.

---

## Состав отчёта (BR-F-1008)

| Раздел | Описание |
|---|---|
| По моделям | Количество единиц каждой модели в парке |
| По клиентам | Количество единиц оборудования у каждого клиента |
| По возрасту | Распределение: до 1 года / 1–3 / 3–5 / более 5 лет |
| ТОП-10 проблемных | Единицы с наибольшим количеством ремонтов за период |

## Acceptance Criteria

```gherkin
Feature: Отчёт по парку оборудования

  Scenario: Формирование отчёта
    Given в системе 50 единиц оборудования у 10 клиентов
    When Директор формирует отчёт за 2026 год
    Then отображаются все 4 раздела отчёта с корректными данными

  Scenario: ТОП-10 проблемных единиц
    Given оборудование SN-001 имеет 8 ремонтов за период — больше всех
    When формируется отчёт
    Then SN-001 занимает первое место в ТОП-10
```
