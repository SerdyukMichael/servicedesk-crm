# UC-1006: Сформировать отчёт по парку оборудования

**Версия:** 1.0 | **Дата:** 26.03.2026 | **Статус:** На проверке

---

## Паспорт

| Поле | Значение |
|---|---|
| **ID** | UC-1006 |
| **Модуль** | Модуль 10: Учёт оборудования |
| **Основной актор** | Директор |
| **Второстепенные акторы** | Руководитель сервиса |

| **Постусловия** | Отчёт сформирован; доступен для просмотра и экспорта |

---

## Основной поток

1. Пользователь открывает раздел отчётов → «Отчёт по парку оборудования».
2. Задаёт период анализа (диапазон дат).
3. Нажимает **«Сформировать»**.
4. Система формирует отчёт и отображает разделы.
5. Пользователь экспортирует в PDF или XLSX при необходимости.

---

## Состав отчёта (BR-F-1008)

| Раздел | Описание |
|---|---|
| По моделям | Количество единиц каждой модели в парке |
| По клиентам | Количество единиц оборудования у каждого клиента |
| По возрасту | Распределение: до 1 года / 1–3 / 3–5 / более 5 лет |
| ТОП-10 проблемных | Единицы с наибольшим количеством ремонтов за период |

---

## Альтернативные потоки

**АП-1: Фильтрация по клиенту или модели**
Пользователь выбирает клиента или модель оборудования до нажатия «Сформировать».
Система строит отчёт только по выбранному подмножеству.

**АП-2: Экспорт сформированного отчёта**
После формирования пользователь нажимает «Экспорт PDF» или «Экспорт XLSX».
Система генерирует файл с заголовком, датой формирования и всеми разделами.

---

## Ключевые данные (состав отчёта)

| Раздел | Поля | Источник |
| --- | --- | --- |
| По моделям | model_name, count | equipment |
| По клиентам | client_name, count | equipment JOIN clients |
| По возрасту | age_bucket, count | equipment.commissioned_at |
| ТОП-10 проблемных | serial_number, model, repair_count | service_requests (closed) |

---

## Исключения

**ИП-1: Нет данных за выбранный период**
→ Система отображает: *«За выбранный период данных нет»*. Разделы отчёта пусты, экспорт недоступен.

**ИП-2: Некорректный диапазон дат (начало > конец)**
→ Кнопка «Сформировать» неактивна; подсказка: *«Дата начала не может быть позже даты окончания»*.

## Acceptance Criteria

```gherkin
Feature: Отчёт по парку оборудования

  Scenario: Формирование отчёта
    Given в системе 50 единиц оборудования у 10 клиентов
    When Директор формирует отчёт за 2026 год
    Then отображаются все 4 раздела отчёта с корректными данными

  Scenario: ТОП-10 проблемных единиц
    Given оборудование SN-001 имеет 8 ремонтов за период — больше всех
    When формируется отчёт
    Then SN-001 занимает первое место в ТОП-10

  Scenario: Экспорт в XLSX
    Given отчёт сформирован
    When пользователь нажимает «Экспорт XLSX»
    Then скачивается файл с четырьмя листами (по одному на раздел)

  Scenario: Нет данных за период
    When Директор задаёт период без оборудования и нажимает «Сформировать»
    Then отображается «За выбранный период данных нет»
    And кнопка «Экспорт» неактивна

  Scenario: Фильтрация по клиенту
    Given в системе оборудование 5 клиентов
    When пользователь выбирает «Банк Альфа» и формирует отчёт
    Then все разделы содержат только оборудование «Банк Альфа»
```
