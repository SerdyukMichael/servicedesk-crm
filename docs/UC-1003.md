# UC-1003: Управление курсами валют

**Версия:** 1.1 | **Дата:** 21.04.2026 | **Статус:** Рабочий

---

## Паспорт

| Поле | Значение |
|---|---|
| **ID** | UC-1003 |
| **Модуль** | Модуль 12: Системные настройки |
| **Основной актор** | Администратор, Бухгалтер |
| **Второстепенные акторы** | Все авторизованные пользователи (просмотр) |
| **Связанные BR-F** | BR-F-102 |
| **Приоритет** | Must |
| **Предусловия** | Пользователь авторизован с ролью **admin** или **accountant**; в системе задана системная валюта |
| **Постусловия** | Новый курс сохранён в историю; запись с наибольшим `set_at` по валюте считается актуальным курсом |
| **Триггер** | Пользователь открывает «Настройки → Курсы валют» и нажимает «Установить курс» |

---

## Основной поток — Установка курса

1. Пользователь открывает **«Настройки → Курсы валют»**.
2. Система отображает таблицу актуальных курсов (запись с наибольшим `set_at` по каждой валюте): иностранная валюта, курс к системной валюте, дата установки, кто установил.
3. Пользователь нажимает **«Установить курс»**.
4. Система открывает форму с полями:
   - **Иностранная валюта** — код ISO 4217 (например: USD, EUR); ввод вручную
   - **Курс** — сколько единиц системной валюты за 1 единицу иностранной; decimal > 0
   - **Дата курса** — дата и время установки курса; по умолчанию — текущий момент; допускается указать прошедшую или будущую дату
5. Пользователь заполняет форму и нажимает **«Сохранить»**.
6. Система создаёт новую запись в `exchange_rates` с `set_by = current_user`, `set_at = указанная дата (или now() если не указана)`.
7. Обновлённый актуальный курс отображается в таблице.

---

## Альтернативные потоки

### А1 — Некорректный код валюты
- Шаг 5: код не соответствует формату (не 3 заглавные латинские буквы).
- Система показывает ошибку валидации, не сохраняет запись.

### А2 — Некорректный курс
- Шаг 5: курс ≤ 0 или не является числом.
- Система показывает ошибку валидации, не сохраняет запись.

### А3 — Дата курса в прошлом

- Шаг 5: пользователь указывает прошедшую дату (например, задним числом).
- Система принимает запись, сохраняет `set_at` с указанной датой.
- Актуальным курсом остаётся запись с наибольшим `set_at` (старая запись задним числом не вытесняет более позднюю).

### А4 — Дата курса в будущем

- Шаг 5: пользователь указывает будущую дату.
- Система принимает запись, сохраняет `set_at` с указанной датой.
- Когда наступит указанная дата, эта запись становится актуальной (сортировка `set_at desc`).

---

## Основной поток — Просмотр истории

1. Пользователь выбирает иностранную валюту из таблицы актуальных курсов.
2. Система отображает историю записей по выбранной валюте: дата, курс, кто установил — сортировка по убыванию `set_at` (при равенстве — по убыванию `id`).

---

## Бизнес-правила

| ID | Правило |
|---|---|
| BR-R-201 | Код иностранной валюты — строго 3 заглавные латинские буквы (ISO 4217) |
| BR-R-202 | Курс должен быть положительным числом (> 0) |
| BR-R-203 | Каждая установка создаёт новую запись; старые записи не изменяются и не удаляются |
| BR-R-204 | Актуальный курс — запись с наибольшим `set_at` по каждой иностранной валюте; при равенстве `set_at` — запись с большим `id` |
| BR-R-205 | Установить курс могут только роли `admin` и `accountant`; просматривать — все авторизованные |
| BR-R-206 | Поле `set_at` при установке курса опционально: если не передано, сервер подставляет текущий момент; допускается любая дата (прошедшая или будущая) |

## Acceptance Criteria

```gherkin
Feature: Курсы валют

  Scenario: admin устанавливает курс USD с датой по умолчанию
    Given admin авторизован
    When он устанавливает курс USD = 450.0000 без указания даты
    Then новая запись появляется в таблице актуальных курсов
    And set_at записи приблизительно равен текущему моменту

  Scenario: admin устанавливает курс задним числом
    Given admin авторизован
    When он устанавливает курс USD = 440.0000 с set_at = 2025-01-01T00:00:00
    Then запись сохраняется с set_at = 2025-01-01T00:00:00
    And актуальным курсом USD остаётся запись с наибольшим set_at

  Scenario: admin устанавливает курс будущей датой
    Given admin авторизован
    When он устанавливает курс USD = 460.0000 с set_at = 2099-12-31T00:00:00
    Then запись сохраняется с set_at = 2099-12-31T00:00:00
    And эта запись становится актуальной (наибольший set_at)

  Scenario: engineer не может установить курс
    Given engineer авторизован
    When он отправляет POST /api/v1/settings/exchange-rates
    Then система отвечает 403 Forbidden

  Scenario: валидация кода валюты
    Given admin авторизован
    When он отправляет код "usd" (строчные буквы)
    Then система отвечает 422 Unprocessable Entity

  Scenario: история отсортирована по set_at desc
    Given в системе есть записи курса EUR с разными датами
    When любой авторизованный пользователь запрашивает историю EUR
    Then записи возвращаются в порядке убывания set_at
```
